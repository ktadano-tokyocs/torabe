職員共有資料 ／ 写真・お知らせ・説明会を自力で更新するための手引き ／ 作成日：2026年6月25日

とらべのサイトは 1枚の静的HTML で動いています。日々の更新は 2か所だけ 触れば完結し、HTMLを編集する必要はありません。①特長セクションの写真は WordPressメディア（同じファイル名のまま上書き）、②お知らせ・説明会は Googleスプレッドシート（編集すると自動でサイトに反映）で管理しています。

1特長セクションの写真を差し替える

「とらべが選ばれる4つの特長」の写真は、WordPressメディアの固定URLで配信しています。同じファイル名のまま中身だけ上書きすれば、HTMLを触らずにサイトへ反映されます。

特長ファイル名配信URL（2026/06フォルダ）
1. 日本語feature1-japanese.jpg…/uploads/2026/06/feature1-japanese.jpg
2. 探究feature2-inquiry.jpg…/uploads/2026/06/feature2-inquiry.jpg
3. 原体験feature3-experience.jpg…/uploads/2026/06/feature3-experience.jpg
4. 個別評価feature4-evaluation.jpg…/uploads/2026/06/feature4-evaluation.jpg

URLの先頭は共通：https://tokyocs.org/wordpress/wp-content/uploads/2026/06/

差し替え手順

  1. WordPress管理画面にログインし、メディアを開く
  2. 差し替えたい画像（例：feature1-japanese.jpg）を選択する
  3. 編集画面の 「Enable Media Replace（メディアを置換）」 をクリック
  4. 「ファイルを置換（ファイル名はそのまま）」 を選び、新しい画像をアップロードして保存
  5. サイトを開いて、写真が変わっていることを確認
⚠ ここだけ注意 必ず「ファイル名はそのまま」モードを選んでください。もう一方の「新しいファイル名にして更新」を選ぶとURLが変わり、サイトに表示されなくなります。拡張子も .jpg のままにしてください。
✓ きれいに見せるコツ 横長（縦横比 4:3〜16:9 程度）／ サイズの目安は横1200px前後。大きすぎる画像は表示が重くなります。

反映されないとき：ブラウザのスーパーリロード（Mac Shift+⌘+R ／ Windows Ctrl+F5）で再読み込み。WordPress側のキャッシュは最新版プラグインが自動クリアします。

2お知らせ・説明会を更新する（スプレッドシート）

「お知らせ」と「説明会」は Googleスプレッドシート で管理しています。シートを編集して保存すると、自動的にサイトへ反映されます（サイト側の作業は不要）。下の表のとおり各列に入力し、公開列を TRUE にすれば表示、FALSE なら下書き（非表示）です。

「お知らせ」シート

列内容

Bタイトル
C概要（任意）
DリンクURL（任意・空欄可）
E公開 TRUE / FALSE

「説明会」シート

列内容
A日付（例 2026/06/16）
B時間（例 19:00〜20:30）
C形式（オンライン／現地）
D公開 TRUE / FALSE
－（お知らせより1列少ない）
  1. シートに行を追加、または既存の行を編集する
  2. 表示したい行は公開列を TRUE にする（下書きは FALSE）
  3. 保存すれば自動でサイトに反映されます
ⓘ 覚えておくこと お知らせは新しい日付が上に表示されます。データを更新するだけなら再デプロイは不要です（プログラム news.gs 自体を変えたときだけ再デプロイが必要）。

第1章の写真更新に必要なプラグインについて（導入のお願い）

WordPress管理者・決裁者の方へ ／ 「Enable Media Replace」導入の許可をお願いする資料です

3なぜ「Enable Media Replace」が必要なのか

第1章の写真差し替えは、同じファイル名のまま中身だけを上書きすることで、サイトのHTMLを一切触らずに写真を更新できる仕組みです。ところが、WordPressの標準機能には「ファイル名を変えずに上書きする」機能がありません。標準のやり方で画像をアップロードすると毎回ファイル名（=URL）が変わってしまい、その都度サイトのHTMLを書き換える作業が発生します。

これを解決するのが、写真を安全に上書きするための定番プラグイン 「Enable Media Replace」 です。導入をご許可いただけますようお願いいたします。

このプラグインで得られること

✓ 非エンジニアの職員だけで更新が完結する 導入後は、職員が管理画面のボタン操作だけで写真を差し替えられます。外注や開発者を待つ必要がなく、思い立ったその場で最新の写真に更新できます。HTMLの知識は一切不要です。
✓ 安全・信頼できる（公式・無料・実績）
  • WordPress公式ディレクトリに掲載された無料プラグインで、有効インストール数が非常に多く、長期間メンテナンスされ続けている定番です。
  • 役割は「メディアの置換」だけに限定されており、サイトの他の部分には影響しません。
  • URLを変えずに安全に上書きでき、最新版は主要なキャッシュも自動でクリアします。
ⓘ 導入しない場合に起きること 写真を更新するたびにURLが変わるため、毎回エンジニアによるHTML編集（サイト内4か所のURL書き換え）が必要になり、手間とコストがかかり続けます。

導入手順（管理者の作業・数分で完了）

  1. WordPress管理画面 → プラグイン → 新規追加 を開く
  2. 検索欄に 「Enable Media Replace」 と入力して検索
  3. 今すぐインストール → 有効化 をクリック
✓ 導入のお願い（まとめ） 公式・無料・実績豊富で、権限はメディア置換に限定。導入により職員だけで写真更新が完結し、運用の手間とコストを継続的に削減できます。上記3ステップでの有効化をご許可ください。

困ったとき（トラブルシュート）

  • 特長の写真が表示されない：URLが https:// になっているか／ファイル名・拡張子（.jpg）・月フォルダ（2026/06）が一致しているか／その画像がメディアに存在するか。
  • お知らせ・説明会が表示されない：公開列が TRUE になっているか／スプレッドシートを保存したか（反映に少し時間がかかる場合あり）。
